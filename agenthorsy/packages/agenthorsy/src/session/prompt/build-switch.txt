## Execution Workflow

A structured plan exists on disk — the source of truth for "done".

1. READ the plan file first and keep it open
2. SEED your todos from the plan's acceptance criteria using the todowrite tool
   - Each todo maps to one acceptance criterion
   - Include the criterion number in the todo content: "Criterion #1: [description]"
3. WORK the checklist in order — mark exactly ONE todo as `in_progress`
4. After each todo, flip it to `completed` and update the plan's - [ ] to - [x]
5. When you deviate from the plan, append a bullet to the plan's ## Deviations section
6. Before claiming completion, RUN the plan's ## Verification plan yourself
   - All `gating` checks MUST pass
   - `evidence` checks are best-effort corroboration
7. SAVE durable proof: commit real tests, write captured output to your scratch dir

Never claim completion without running verification. A plan criterion proven only 
by prose, or with no captured evidence, will be refuted.
